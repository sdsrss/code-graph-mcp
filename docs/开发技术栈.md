选择 **路线 B（Rust 极致单体）**，意味着你将打造出一个开源界的“神级工具”：**零外部依赖、极低内存占用、单体二进制文件（约 50MB-80MB，内嵌微型 AI 模型），真正做到“插上就能用”**。

下面是为你量身定制的 **Rust 核心开发蓝图与架构落地方案**：

## 核心依赖清单 (`Cargo.toml` 选型)

要实现这个复杂的系统且保持单体，我们需要挑选 Rust 生态中最硬核的库：

- **MCP 通信**：`mcp-sdk` 或纯手工实现基于 `stdio` 的 JSON-RPC 2.0（非常简单，用 `serde_json` 即可）。
- **本地 AI 推理 (替代 Ollama)**：`candle-core` 和 `candle-nn`（HuggingFace 官方的纯 Rust 机器学习框架），用于在内存中极速加载小型 Embedding 模型。
- **数据库与搜索**：`rusqlite`（开启 `bundled` 和 `fts5` 特性） + 在 `build.rs` 中用 `cc` 静态编译打包 `sqlite-vec` C 语言源码。
- **代码解析**：`tree-sitter` 及各语言的 Rust binding（`tree-sitter-javascript`, `tree-sitter-go` 等）。
- **增量与文件系统**：`notify`（跨平台毫秒级文件监听），`blake3`（目前最快的加密哈希，用于生成 Merkle Tree），`ignore`（自动解析 `.gitignore`）。

## 架构设计与模块划分

## 1. 知识图谱与向量数据库 (SQLite Schema 设计)

使用单一的 `index.db` 文件（存放在项目根目录的 `.code-graph/` 下），建立以下核心表结构：

```
sql-- 1. Merkle Tree 与增量缓存表
CREATE TABLE files (
    path TEXT PRIMARY KEY,
    blake3_hash TEXT NOT NULL,
    last_modified INTEGER
);

-- 2. 知识图谱：节点表 (FTS5 全文搜索)
CREATE VIRTUAL TABLE nodes USING fts5(
    id UNINDEXED,         -- UUID
    type,                 -- 'function', 'class', 'route'
    name,                 -- 函数名/类名 (如 'authUser')
    code_content,         -- 源码文本
    context_string        -- 图谱增强文本 (用于生成向量的组合文本)
);

-- 3. 知识图谱：边关系表
CREATE TABLE edges (
    source_id TEXT,       -- 调用者
    target_id TEXT,       -- 被调用者
    relation_type TEXT    -- 'calls', 'inherits', 'routes_to'
);

-- 4. sqlite-vec 向量表
CREATE VIRTUAL TABLE node_vectors USING vec0(
    node_id TEXT PRIMARY KEY,
    embedding float[384]  -- 适配 all-MiniLM-L6-v2 的 384 维向量
);
```

## 2. “完全零配置” 的本地 AI 引擎

这是最硬核的一步：在 Rust 程序里直接运行模型。

- **模型选择**：选择 `all-MiniLM-L6-v2` 的量化版（只有约 20-30MB 大小）。
- **加载方式**：在编译时使用 `include_bytes!` 宏将 `.safetensors` 模型文件直接打包进 Rust 二进制文件（`.exe` / 可执行程序）中。
- **执行流程**：系统启动时，`candle` 框架将这 30MB 载入内存。当有代码需要嵌入时，直接在 Rust 进程内计算，**耗时只需几毫秒，不经过任何网络栈，连本地 TCP 端口都不用开！**

## 3. 虚拟化上下文 (Context Mode) 防爆机制

为解决 Claude Token 超限问题，实现类似 `mksglu/context-mode` 的机制：

- 在 SQLite 中增加一张 `context_sandbox` 表。
- 当 MCP 执行混合搜索查找到 5000 行关联代码时，系统**不直接**返回代码。
- 而是将这 5000 行存入沙盒，给 Claude 返回：`{"summary": "找到了 authUser 等 5 个核心鉴权函数", "pointers": ["sandbox_id_1", "sandbox_id_2"]}`。
- 提供一个 `read_snippet(pointer)` 的 MCP Tool，让 Claude 像翻书一样，想看哪段再调哪段。

## 开发攻坚路径 (分四个 Sprint)

**Sprint 1: 骨架与 MCP 协议通信 (第 1-3 天)**

- 搭建 Rust 项目，实现基于标准输入/输出的 MCP Server 通信。
- 向 Claude Code 注册第一个测试 Tool：`ping`。确保 Claude 能正确调用。

**Sprint 2: AST 图谱解析与 SQLite 存储 (第 4-7 天)**

- 引入 `tree-sitter`，编写针对 TS/JS 或 Python 的查询语法（Tree-sitter Queries），提取函数名和调用关系。
- 实现 `get_call_graph(func_name)` 工具：使用 SQLite 的 **递归 CTE (WITH RECURSIVE)** 瞬间查出某个函数的完整调用链路。

**Sprint 3: 本地 AI 与混合搜索 (第 8-12 天)**

- 集成 `candle` 运行文本到向量的模型。
- 把解析出的图谱上下文（`路由 /api/login -> authUser 函数 -> db.query`）拼接后生成向量，写入 `sqlite-vec` 表。
- 实现 `semantic_search(query)` 工具：内部将用户的自然语言 query 转为向量，利用 FTS5 (BM25) 与 Vec0 进行双路合并得分 (RRF)。

**Sprint 4: 实时增量与极致优化 (第 13-14 天)**

- 挂载 `notify` 监听文件保存事件。
- 使用 `blake3` 计算文件 Hash 树。如果只改了一行，只让那一行的函数重新生成向量，实现毫秒级“无感同步”。

## Claude Code 最终接入与发布形态

开发完成后，通过 GitHub Actions 使用跨平台编译（`cargo cross`），编译出 `windows-x64.exe`、`macos-arm64`、`linux-x64` 三个独立文件（每个约 60MB，内含数据库引擎和 AI 模型）。

**用户的使用体验将是极其震撼的：**

1. 用户不需要配置 Docker，不需要下载 Ollama，不需要提供 API Key。

2. 在终端运行一行命令（例如借助 npx 分发二进制）：

   ```
   bash
   claude mcp add code-graph-mcp npx @your-name/code-graph-mcp
   ```

3. 用户在 Claude Code 中输入：“帮我查一下登录鉴权的完整调用链路出什么 Bug 了？”

4. 你的 Rust MCP 在后台以 **30ms** 的速度算出增量，以 **10ms** 的速度查出调用图谱，Claude 瞬间定位 Bug，只消耗了不到 200 个 Token。

这个方案难度在于 `candle` 和 `sqlite-vec` 的 Rust 静态编译集成。你可以先从 **Sprint 1 的 MCP 通信** 或者 **用 `tree-sitter` 在 Rust 中提取出一个 JS 函数的调用关系** 开始，你想先看哪部分的核心代码逻辑？
