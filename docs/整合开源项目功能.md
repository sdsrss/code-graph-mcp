这五个开源项目确实代表了目前代码索引 MCP 领域的最高水准。如果我们要把它们**最核心、最惊艳的“杀手级功能”**全部抽离出来，揉捏进我们那个“单体 Go 二进制 + SQLite”的超级方案中，这绝对是一个极客级别的梦幻联动。

下面是我们要从这 5 个项目中“偷师”并整合的**核心优秀功能清单与落地设计**：

## 1. 偷师 `zilliztech/claude-context`

- **优秀功能提取：基于 Merkle Tree 的毫秒级增量索引与 Hybrid Search (混合搜索)**。
- **整合方案**：
  - **Merkle Tree 极速增量**：不每次扫描几万个文件，而是给每个文件夹算一个 Hash。按 `Ctrl+S` 保存时，只比对 Hash 变的那个文件。Zilliz 用它实现了超大库的极速同步，我们用 Go 语言将它写死在本地，速度比 Node.js 更快。
  - **混合搜索 (Hybrid Search)**：Zilliz 最大的卖点是 `BM25 词频匹配 + 向量相似度 = 顶级准确率`。我们在 SQLite 中直接开通 **FTS5 (BM25全文搜索)** 和 **sqlite-vec (向量搜索)**，并使用 RRF 算法在本地完成双路召回合并。既能搜“登录意图”，也能精确匹配变量名 `auth_user_id`。

## 2. 偷师 `DeusData/codebase-memory-mcp`

- **优秀功能提取：代码知识图谱 (Knowledge Graph) 与 99% 的 Token 节省**。
- **整合方案**：
  - 这个用 Go 写的神级项目证明了：**让大模型看关系图谱，比让它看几千行源码强百倍！**
  - 我们整合它的 Tree-sitter 图谱引擎。解析出 `类 A 继承了 B`，`路由 /api/user 映射到 getUser 函数`。
  - 提供给 Claude 一个 `get_call_chain` (获取调用链) 工具。当排查 Bug 时，Claude 只需花费 50 个 Token 拿到整个微服务调用的骨架，瞬间理清逻辑，而不用阅读几十 KB 的源码。

## 3. 偷师 `johnhuang316/code-index-mcp`

- **优秀功能提取：极致的“开箱即用”与智能屏蔽 (`node_modules` 过滤)**。
- **整合方案**：
  - 放弃复杂的配置。像 John Huang 的项目一样，系统启动时自动读取 `.gitignore`、`.npmignore`、`.dockerignore`，自动避开 `dist`、`build`、`node_modules` 等垃圾目录。
  - **精确符号查找 (Exact Symbol Search)**：除了向量搜索，必须保留传统的“跳转到定义”功能。当 Claude 明确知道要找 `UserController` 时，直接利用 AST 提取，1 毫秒内把这个类的源码原封不动地抛给 Claude，不经过任何大模型推测。

## 4. 偷师 `mksglu/context-mode`

- **优秀功能提取：沙盒拦截与 98% 的上下文压缩 (Context Virtualization)**。
- **整合方案**：
  - `context-mode` 的核心思想是**“别让工具把大模型的脑子塞满”**。当执行复杂的搜索或命令时，如果返回了 300KB 的日志或代码，Claude 会瞬间变傻（Context Window 被填满）。
  - 我们整合它的**虚拟化拦截思想**：当搜索返回匹配到的长篇代码时，我们的 MCP 不直接把 5000 行代码扔给 Claude，而是**返回一个摘要和“虚拟指针”**（例如：`找到 5 个匹配项，已存入沙盒，请调用 read_snippet(id) 按需阅读`）。这能把 315KB 的冗余输出压缩到 5.4KB，让 Claude 连续聊 3 个小时都不会出现“上下文超限”的卡顿。

## 5. 偷师 `danielbowne/claude-context`

- **优秀功能提取：零依赖本地向量存储与纯本地大模型接入 (Ollama/ONNX)**。
- **整合方案**：
  - 原版 Zilliz 需要连云端 Milvus 数据库，而 Daniel 的分支证明了单机向量库（LanceDB）的可行性。
  - 我们走得更极端：**连 LanceDB 都不用**。直接使用打包在单体文件里的 `sqlite-vec`。
  - **本地向量计算**：支持一键接入本地 Ollama 的 `nomic-embed-text`，或者直接在 Go 二进制程序里内嵌一个 ONNX 引擎跑微型 Embedding 模型。你的代码连局域网都不出。

## 🚀 终极产品形态勾勒：神级本地 MCP

如果我们把这些特性完美揉合成一个名为 **`code-graph-mcp`** 的 Go 语言二进制文件：

1. **安装体验**：只有 1 个文件。你在 Claude Code 里输入 `/mcp add code-graph-mcp` 即可，0 外部依赖。
2. **启动速度**：基于 Go + SQLite，1 秒内吃透 5 万行代码的知识图谱并建立向量与 Merkle Tree。
3. **Claude 的超能力**：
   - Claude 遇到长代码时，能自动触发 **Context-Mode 的压缩机制**，绝不浪费 Token。
   - 找寻复杂 Bug 时，Claude 会调用图谱工具（**DeusData 附体**），10 毫秒拿到跨文件的 HTTP 调用链。
   - 找寻模糊语义时，触发 **Hybrid Search（Zilliz 附体）**，通过本地模型（**Daniel 附体**）瞬间双路召回精准代码块。
